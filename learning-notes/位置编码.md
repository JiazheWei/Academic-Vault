
# Transformer的位置编码
我们知道attn机制是无法区分不同位置的token的，因此让attn机制有位置意识的方法大体有两种：
- 将位置信息融入到输入中，这是**绝对位置编码**的做法；
- 微调attn机制的结构，让它有办法区分不同位置的token，这是**相对位置编码**的做法。

# 绝对位置编码
在输入的第$k$个向量$x_{k}$加上向量$p_{k}$， 构成$x_k+p_k$， $p_k$的值仅由位置$k$决定。

# 可训练的绝对位置编码
GPT， bert等都采用这种编码方式。绝对位置编码一个最朴素的想法是不去设计什么，也就是我们不事先约定刚才说到的$p_k$这些向量的值是多少，我们把这些向量当做可学习的参数跟着模型一起训就行了。具体地，如果序列长度上限为512，$dim=768$，那我们约定位置编码矩阵就是一个$512 \times 768$ 的参数矩阵就行了，训练的时候不断更新这个矩阵。

缺陷很明显，当序列长度超过512的时候这种位置编码就不适用了。但后来发现，用[层次分解](https://spaces.ac.cn/archives/7947)的方式能缓解这个问题，所以长度缺陷也不是很明显。

# 三角式位置编码
attn is all you need 这篇文章提出来的绝对位置编码显式最优解：
$$
\begin{cases}
\boldsymbol{p}_{k, 2i} = \sin \left( k / 10000^{2i/d} \right) \\
\boldsymbol{p}_{k, 2i+1} = \cos \left( k / 10000^{2i/d} \right)
\end{cases}
\tag{1}
$$
其中$p_{k,i}$表示位置是$k$的编码向量的第$i$个分量。自attn is all you need之后这种三角位置编码见得不多。

# 旋转位置编码

## 概念

[知乎](https://zhuanlan.zhihu.com/p/647109286)
假定现在词嵌入向量的维度是两维 $d=2$，这样就可以利用上2维度平面上的向量的几何性质，然后论文中提出了一个满足上述关系的 $f$ 和 $g$ 的形式如下：

$$
\begin{aligned}
f_q(\boldsymbol{x}_m, m) &= (\boldsymbol{W}_q \boldsymbol{x}_m) e^{im\theta} \\
f_k(\boldsymbol{x}_n, n) &= (\boldsymbol{W}_k \boldsymbol{x}_n) e^{in\theta} \\
g(\boldsymbol{x}_m, \boldsymbol{x}_n, m-n) &= \text{Re} \left[ (\boldsymbol{W}_q \boldsymbol{x}_m) (\boldsymbol{W}_k \boldsymbol{x}_n)^* e^{i(m-n)\theta} \right]
\end{aligned}
\tag{8}
$$

这里面 $\text{Re}$ 表示复数的实部。

进一步地，$f_q$ 可以表示成下面的式子：

$$
\begin{aligned}
f_q(\boldsymbol{x}_m, m) &= \begin{pmatrix} \cos m\theta & -\sin m\theta \\ \sin m\theta & \cos m\theta \end{pmatrix} \begin{pmatrix} W_q^{(1,1)} & W_q^{(1,2)} \\ W_q^{(2,1)} & W_q^{(2,2)} \end{pmatrix} \begin{pmatrix} x_m^{(1)} \\ x_m^{(2)} \end{pmatrix} \\
&= \begin{pmatrix} \cos m\theta & -\sin m\theta \\ \sin m\theta & \cos m\theta \end{pmatrix} \begin{pmatrix} q_m^{(1)} \\ q_m^{(2)} \end{pmatrix}
\end{aligned}
\tag{9}
$$

看到这里会发现，这不就是 query 向量乘以了一个旋转矩阵吗？这就是为什么叫做旋转位置编码的原因。

## 实现
计算频率的方式如下：
```
   # 计算每个维度对应的频率: 1 / (theta^(2i/d))

    # torch.arange(0, dim, 2) 生成 [0, 2, 4, ..., dim-2]，取前 dim//2 个

    freqs = 1.0 / (theta ** (torch.arange(0, dim, 2)[: (dim // 2)].double() / dim))
```

其中i 指的是处于第几个二元组。

假设token的dim=8，那么：
```
Token的8个维度分成4对：(d0,d1), (d2,d3), (d4,d5), (d6,d7)
                        ↓        ↓        ↓        ↓
对应的频率：           freq_0   freq_1   freq_2   freq_3
                      (高频)                      (低频)
                       1.0      0.1      0.01     0.001
```

举个例子看pos=3的token的位置编码旋转角的计算方法就是：
```
原始token (位置3):  [a,   b,   c,   d,   e,   f,   g,   h]
                    └─┘       └─┘       └─┘       └─┘
                   第0对     第1对     第2对     第3对

旋转角度:          3×1.0    3×0.1    3×0.01   3×0.001
                   = 3.0    = 0.3    = 0.03   = 0.003
                   (转很多)          ...      (几乎不转)

旋转后:            [a',b'] [c',d'] [e',f'] [g',h']

其中 [a',b'] = 把(a,b)旋转3.0弧度
     [c',d'] = 把(c,d)旋转0.3弧度
     ...
```

这时候就得到了旋转角矩阵，再按照r=1的极坐标将旋转角矩阵转换为相应的复数矩阵，再让Q K分别与对应的复数矩阵相乘即可。

完整流程：
```
Step 1: 计算频率向量（dim//2 个频率）
        freqs = [1.0, 0.1, 0.01, 0.001]
                 ↓
Step 2: 外积得到旋转角度矩阵 (第108行)  ← 你问的这行
        torch.outer(positions, freqs)
        结果: (end, dim//2) 的角度矩阵
                 ↓
Step 3: 角度转复数 (第111行)
        torch.polar(1, 角度) → e^(iθ)
        结果: (end, dim//2) 的复数矩阵
                 ↓
Step 4: 应用到Q和K (rope_apply函数)
        Q × 复数 = 旋转后的Q
        K × 复数 = 旋转后的K
```

矩阵中每一项的实部和复部最后分别拆到两个维度，正好将矩阵从dim//2维拽回dim维。

处理视频用的3D rope与1D rope类似，唯一的区别在于：
1) 现在f,w,h 三个维度要共用原来那么多dim，例如recammaster中设置了22+21+21=128/2的分配方式。
2) 1D位置编码只需要一个pos值就能定位这个token对应的位置编码是什么，3Drope需要f,w,h才能确定一个token的位置，自然位置编码也就是3个freq矩阵拼接的结果。

具体来说：
首先按照1D rope一样算出三个维度的freq矩阵：
```
# 模型初始化时预计算（head_dim = 128）
self.freqs = precompute_freqs_cis_3d(head_dim=128)

返回:
self.freqs[0]: [1024, 22] (complex64)  # 帧维度，44实数→22复数
self.freqs[1]: [1024, 21] (complex64)  # 高度维度，42实数→21复数  
self.freqs[2]: [1024, 21] (complex64)  # 宽度维度，42实数→21复数

注: 22 + 21 + 21 = 64 (复数) = 128 (实数) / 2
```

假设当前video的patch grid是：
```
f = 16  (帧数)
h = 128 (高度方向的patch数)
w = 128 (宽度方向的patch数)
```

那就在三个矩阵中取对应的部分：
```
self.freqs[0][:f]  # 取前16个帧位置
self.freqs[1][:h]  # 取前128个高度位置
self.freqs[2][:w]  # 取前128个宽度位置
```

之后依次处理帧，高，宽维度频率矩阵：

```
#帧维度的：
[16, 22] (complex)
   ↓ view(16, 1, 1, 22)
[16, 1, 1, 22] (complex)
 ↑   ↑  ↑  ↑
 f   1  1  频率维度

   ↓ expand(16, 128, 128, 22)
[16, 128, 128, 22] (complex)
 ↑   ↑    ↑    ↑
 f   h    w   频率

含义: 同一帧内的所有(h,w)位置共享同一个帧位置编码
例如: 
  位置(f=0, h=任意, w=任意) 都用 freqs[0][0]
  位置(f=5, h=任意, w=任意) 都用 freqs[0][5]
  
#高维度的：
[128, 21] (complex)
   ↓ view(1, 128, 1, 21)
[1, 128, 1, 21] (complex)
 ↑  ↑    ↑  ↑
 1  h    1  频率维度

   ↓ expand(16, 128, 128, 21)
[16, 128, 128, 21] (complex)
 ↑   ↑    ↑    ↑
 f   h    w   频率

含义: 同一行(h)内的所有(f,w)位置共享同一个高度位置编码
例如:
  位置(f=任意, h=10, w=任意) 都用 freqs[1][10]
  位置(f=任意, h=50, w=任意) 都用 freqs[1][50] 
  
# 宽维度的：
[128, 21] (complex)
   ↓ view(1, 1, 128, 21)
[1, 1, 128, 21] (complex)
 ↑  ↑  ↑    ↑
 1  1  w   频率维度

   ↓ expand(16, 128, 128, 21)
[16, 128, 128, 21] (complex)
 ↑   ↑    ↑    ↑
 f   h    w   频率

含义: 同一列(w)内的所有(f,h)位置共享同一个宽度位置编码
例如:
  位置(f=任意, h=任意, w=100) 都用 freqs[2][100]
  位置(f=任意, h=任意, w=50) 都用 freqs[2][50]
  
#最后在dim维度拼接：
torch.cat([
    [16, 128, 128, 22],  # 帧维度RoPE
    [16, 128, 128, 21],  # 高度维度RoPE
    [16, 128, 128, 21]   # 宽度维度RoPE
], dim=-1)  # 在最后一个维度拼接
```

其他显而易见的结论还有Q K的rope矩阵是完全一样的，以及rope是一种不可学习的位置编码。
