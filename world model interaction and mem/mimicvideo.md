## Introduction

### Task and Applications

本文聚焦于**通用机器人操作策略（generalist robot manipulation policy）**的学习问题。具体而言，目标是训练一个以自然语言为条件的策略 π(A_t | o_t, l)，使机器人能够根据语言指令和视觉观测生成动作序列，完成多种操作任务。应用场景涵盖单臂桌面操作、双臂灵巧手操作（如包裹分拣、工具收纳等接触丰富的长时域任务），并在模拟与真实环境中均需具备泛化能力。

### Technical Challenges for Previous Methods

当前主流方法是 Vision-Language-Action Model (VLA)，即在大规模图像-文本预训练的 Vision-Language Model (VLM) 基础上微调以生成机器人动作。这一范式存在如下核心问题：

1. **预训练数据的静态性**：VLM 的预训练数据是静态的图像-文本对，天然缺乏时序上的物理动力学信息（物体如何运动、变形、交互）。因此，VLA 必须在后训练阶段仅从稀缺且昂贵的机器人遥操作数据中**从零学习物理动力学**，导致严重的数据效率瓶颈。
2. **辅助信号的信息瓶颈**：先前工作尝试用视频衍生的辅助信号（如语言计划、affordance、关键点）来增强 VLA 训练，但将稠密视频压缩为稀疏表征会丢失细粒度动力学信息。
3. **先前视频模型方法的计算瓶颈**：之前利用预训练视频模型的方法通常需要学习视频-动作的联合分布，恢复策略时需要在每一个控制步完整生成未来视频帧，计算开销巨大，不适合实时控制。

### 解决 Challenge 的 Pipeline

mimic-video 提出了一种新范式——**Video-Action Model (VAM)**：将一个预训练的互联网规模视频生成模型与一个轻量级动作解码器耦合。Pipeline 的核心思路是：

- 利用视频模型在预训练阶段已经学到的语义和物理动力学先验（而非让策略从零学习）；
- 视频模型在**紧凑的潜空间中做部分去噪**（partial denoising），生成隐式的视觉计划；
- 动作解码器作为一个**逆动力学模型（IDM）**，从视频模型的中间层表征中解码出低维机器人动作。

这样将"预测未来会发生什么"（由视频模型负责）和"将视觉计划翻译为控制信号"（由动作解码器负责）**解耦**，使得动作解码器只需解决一个简单得多的单模态逆动力学问题。

### Contribution 1：提出 Video-Action Model (VAM) 这一新范式

**怎么做的？** 将预训练的视频扩散模型（Cosmos-Predict2, 2B 参数）与基于 Flow Matching 的动作解码器相耦合。视频模型保持冻结，动作解码器通过交叉注意力机制从视频模型的中间层隐状态中提取条件信息来生成动作。

**Key Insight：** 视频预训练天然同时捕获语义和视觉动力学，而 VLM 预训练仅捕获语义。因此，将策略建立在视频模型表征之上，可以使动作解码器的学习任务从"同时学习动力学+控制"简化为"仅学习控制"。这一 insight 通过 oracle 实验得到验证：当用真实未来视频的潜表征来条件化动作解码器时，成功率接近满分，说明控制问题实质上可以被归约为视觉预测问题。

### Contribution 2：部分去噪策略（Partial Denoising Strategy）

**为了解决什么问题？** 在推理时直接从视频模型获取条件信息需要完整的视频生成过程，计算代价高昂；同时完全去噪的视频可能引入生成伪影，导致与训练分布不匹配。

**具体怎么做的？** 引入一个推理时的超参数 τ_v（视频流时间），控制视频去噪的程度。实验发现，在 τ_v = 1（即纯噪声，仅做一次视频 backbone 前向传播）的情况下就能获得最佳平均性能和最快的推理速度。部分去噪的好处有二：(1) 噪声起到数据增强的作用，缓解了预测视频与训练用真实视频之间的分布偏移；(2) 中间去噪步的隐状态编码了比最终去噪状态更丰富的场景动力学信息。

### Contribution 3：10 倍样本效率和 2 倍收敛加速

**为了解决什么问题？** VLA 范式由于需要从稀缺的机器人数据中同时学习动力学和控制，导致数据需求大、训练收敛慢。

**具体怎么做的？** 由于视频 backbone 已经编码了丰富的动力学先验且保持冻结，动作解码器只需学习简单的逆动力学映射。实验结果表明：在 LIBERO 基准上，mimic-video 的动作解码器仅用 10% 的训练数据就达到了 VLA 解码器用全量数据才能达到的最高成功率；即使数据量减少到每任务仅 1 个演示（2% 数据），仍可达到 77% 的平均成功率。此外，收敛速度也快 2 倍以上。

## Method

### Overview

**输入：** 多帧 RGB 图像观测 I_{t-H_o+1}, ..., I_t（观测历史 H_o = 5 帧），自然语言指令 l，以及机器人本体感受状态 q_t。

**输出：** 未来一段动作序列（action chunk）A_t = [a_t, ..., a_{t+H_a-1}]。

**Pipeline 组成：**

1. **视频编码器（3D Tokenizer）**：将 RGB 帧编码为潜空间表征 z。
2. **视频模型（Cosmos-Predict2, 2B DiT）**：在潜空间中基于过去帧和语言指令，对未来帧做（部分）去噪，生成视觉计划。
3. **语言编码器（T5）**：编码语言指令供视频模型做交叉注意力。
4. **动作解码器（轻量级 DiT）**：以视频模型中间层隐状态为条件，结合本体感受状态，通过 Flow Matching 生成动作轨迹。

### Module 1：视频模型（Video Backbone）

**Motivation：** 互联网规模的视频数据天然包含"事情是如何完成的"——物体如何运动、变形、对力做出反应。利用这些预训练知识可以避免从昂贵的机器人数据中重新学习物理动力学。

**Technical Details：** 采用 Cosmos-Predict2 作为基础模型，这是一个 2B 参数的潜空间 Diffusion Transformer (DiT)。输入为 5 帧干净的上下文前缀潜编码和"加噪"的未来帧潜编码的拼接。每个 Transformer 层交替执行：(1) 对完整视频序列的自注意力；(2) 对 T5 编码语言指令的交叉注意力；(3) 两层 MLP。

**为什么能 Work？** 视频模型在预训练中已经学会了丰富的视觉动力学先验。通过 LoRA 微调使其适配机器人视觉域后，其中间层表征就成为了极好的动作解码条件信号。Oracle 实验证实：即使不微调视频模型，仅用真实视频潜表征条件化就能达到近乎满分的成功率。

### Module 2：动作解码器（Action Decoder / Inverse Dynamics Model）

**Motivation：** 需要一个模块将视频模型编码的"视觉计划"翻译为可执行的低维机器人动作。由于视频模型已经处理了多模态的长时域规划，动作解码器只需解决更简单的单模态逆动力学问题。

**Technical Challenge：** 解码器必须对推理时不同的视频噪声水平（τ_v）保持鲁棒，因为训练时和推理时的噪声条件可能不同。

**Technical Details：** 动作解码器也实例化为一个 DiT。它通过两个独立的 MLP 分别编码本体感受状态 q_t 和未来动作序列 A_t，拼接后形成序列维度。每个解码器层包含：(1) 对视频模型中间层表征 h^{τ_v} 的交叉注意力；(2) 对动作序列的自注意力；(3) 两层 MLP。各组件通过残差连接，并使用 AdaLN 进行调制，其中 AdaLN 的输入是视频流时间 τ_v 和动作流时间 τ_a 的低秩双线性仿射编码。训练时随机以学习到的 mask token 替换本体感受状态的 soft token，防止对低维观测的过拟合。

### Module 3：部分去噪与动作采样（Partial Denoising & Action Sampling）

**Motivation：** 完整的视频生成计算代价高昂，且完全去噪的视频可能引入分布偏移。需要一种高效的推理方式。

**Technical Details：** 推理时，从高斯噪声开始对视频流场积分到中间流时间 τ_v，得到部分去噪的潜状态 z^{τ_v}_{future}。将其通过视频模型的前 k 层（k = 19）处理，提取中间层激活作为动作解码器的条件。动作解码器再执行完整的去噪过程生成动作 chunk。当 τ_v = 1 时，视频 backbone 只需一次前向传播（跳过去噪积分步），实现实时推理。

**为什么能 Work？** 两个原因：(1) 噪声作为数据增强，缓解了生成视频与真实视频之间的分布偏移；(2) 去噪中间步的隐状态需要编码丰富的场景动力学变换信息，而接近完全去噪时模型趋向于恒等映射，隐状态反而信息量更少。

### Module 4：两阶段训练（Two-Stage Training）

**阶段一：视频模型微调。** 使用 LoRA 在机器人视频数据上微调视频 backbone，使其适配机器人视觉域的特定语义和动力学，同时保留预训练的时序推理能力。

**阶段二：动作解码器训练。** 冻结视频 backbone，从零训练动作解码器。训练时每次迭代独立采样视频流时间 τ_v（logit-normal 分布）和动作流时间 τ_a（Ta(τ_a) ∝ √τ_a − 0.001），以确保解码器对不同噪声水平的鲁棒性。这种解耦训练方案使得动作解码器的训练非常轻量级和数据高效。

## Experiment

### 资源消耗

- 视频 backbone：Cosmos-Predict2, 2B 参数
- 动作解码器：轻量级 DiT（具体参数量未详述）
- π0.5-style VLA 对比基线：PaliGemma 3B 参数 backbone + 相同的动作解码器
- DiT-Block Policy 基线：约 155M 参数
- 真实世界双臂灵巧操作的动作解码器仅在极少的任务数据上训练：包裹分拣 1 小时 33 分钟（512 episodes），胶带收纳 2 小时 14 分钟（480 episodes）
- 训练超参数：batch size 128-256，AdamW 优化器，学习率 1e-4 至 1.778e-4

### 数据集/Benchmark

1. **SIMPLER-Bridge**（模拟）：基于 BridgeDataV2 数据集训练的 Widow-X 机器人，4 个任务（Put Carrot on Plate, Put Spoon on Towel, Stack Blocks, Eggplant），测试策略在视觉域偏移下的跨任务泛化能力。指标：每任务成功率及平均成功率。对比方法：OpenVLA, Octo, ThinkAct, FLOWER, π0.5-style VLA。
    
2. **LIBERO**（模拟）：Panda 机器人桌面操作，LIBERO-Goal / -Object / -Spatial 三个子集，每任务 50 条专家演示，共 10 个任务。指标：每子集成功率及平均成功率。对比方法：Diffusion Policy, Octo, DiT Policy, OpenVLA, OpenVLA-OFT, π0.5-style VLA。
    
3. **Real-World Dexterous Bimanual**：双臂 Franka Panda + 16-DoF mimic 灵巧手，2 个长时域任务（Package Sorting, Tape Stowing）。指标：成功率。对比方法：DiT-Block Policy（workspace-only vs. multi-view）。
    

### 结果

**SIMPLER-Bridge：** mimic-video 在全部四个任务上取得最高平均成功率 46.9%（scratch 训练），超过所有基线包括使用外部数据微调的 FLOWER（45.0%）和 ThinkAct（43.8%）。通过逐任务调优 τ_v 可进一步提升至 56.3%。特别是在 Eggplant 任务上达到 100% 成功率。

**LIBERO：** mimic-video 以 scratch 训练达到 93.9% 平均成功率，大幅超越同样 scratch 训练的 π0.5-style VLA（85.9%），并超过大多数从通用模型微调的方法（仅 OpenVLA-OFT 的 96.9% 更高，但其使用了外部大规模预训练数据）。

**Real-World Bimanual：** mimic-video 仅使用单个工作空间摄像头就在两项任务上分别达到 72.0% 和 93.0% 的成功率，显著超过使用多视角（含腕部摄像头）的 DiT-Block Policy（42.6% 和 74.1%）。这表明视频模型的预测能力能够有效弥补遮挡带来的视觉不确定性。

**样本效率：** 在 LIBERO 上，mimic-video 用 10% 的数据就达到了 VLA 用 100% 数据的最高性能。即使将数据减少到 2%（每任务仅 1 条演示），仍达到 77% 的成功率，与 Diffusion Policy 基线相当。

**收敛速度：** mimic-video 的动作解码器在训练步数上显著更快地收敛，并达到更高的渐近成功率，尽管 VLA 基线在 FAST 预训练阶段已经接触过任务相关的动作数据。

**τ_v 分析：** 反直觉地，最佳策略性能出现在 τ_v = 1（纯噪声）附近，而非完全去噪。这意味着 mimic-video 在推理时只需视频 backbone 的一次前向传播，同时获得最佳性能和最快推理速度。用真实视频潜表征做的消融实验表明，最低动作重建误差出现在中间流时间 τ_v ≈ 0.4，进一步验证了完全去噪并非必要的结论。
